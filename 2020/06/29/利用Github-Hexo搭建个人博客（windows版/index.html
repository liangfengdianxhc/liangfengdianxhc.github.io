    利用Github-Hexo搭建个人博客（windows版)
    发表于 2020-06-29| 更新于 2020-07-01|有关hexo博客的问题
    阅读量:

    前言

    这篇文章算是本人建立博客以来第一篇正式文章了，请观众老爷多指教。在这几天心血来潮的基础上我去搞了三个博客:一个是csdn的博客、另一个利用阿里云服务器搭建的以wordpress为框架的博客、最后一个就是当前的这个利用Github和Hexo搭建的的个人博客,通过这三次的博客搭建以及使用，我更推荐大家用hexo和github搭建自己的个人博客，主要是有两方面原因：第一个是因为这个不用花钱买服务器和域名，不嫖白不嫖，当然了你也是可以去花一点钱自己弄一个域名也并不是很贵；第二点根据我这几天的使用来说，虽然另外两种的博客使用更方便，但是远没有hexo的这种操作更具有极客范，并且hexo的里面的各种主题十分漂亮，而且如果出现问题网上的解决方案也很丰富。

    总结：hexo+github的这种搭建博客的方式的优点

    • 搭建过程十分简单
    • 操作起来十分帅气
    • 网上面还有很多漂亮的主题
    • 完全免费

    准备过程（建议安装到c盘）

    • 此教程基于windows
    • 安装git（如果链接失效，请去官网下载）
    • 安装node.js（如果链接失效，请去官网下载）

    正式开始搭建hexo博客

    先按win+r，然后输入cmd进入命令窗口（后面的所有指令都是在cmd中执行，只有个别代码是在管理员命令窗口执行后面会强调）

    我们可以先利用cmd里面的node -v和git -v检查一下node.js和git是否安装成功，然后进行下面的步骤

    1. 利用npm指向淘宝镜像源（这里不做过多解释，因为我也不知道具体为啥），输入：

      npm install -g cnpm --registry=https://registry.npm.taobao.org
      然后输入npm如果自动补全，那么说明这个cnpm安装成功了，然后利用cnpm -v检查版本以及是否安装成功

    2. 利用cnpm进行全局安装hexo，输入：

      cnpm install -g hexo-cli
      然后输入hexo -v进行检查hexo是否安装成功

    3. 然后在c盘里手动建立一个文件夹，以后对博客进行更改和检查都是通过这个文件，文件名字随意

    4. 此处用管理员命令窗口执行（如果不会打开管理员命令窗口可以百度方式很多，这里不过多解释）进入你刚刚建立的文件夹目录下输入：

      hexo init（它会自己初始化你的博客框架）
      当出现绿色的INFO后面说start blogging with hexo大概这样子就说明你的hexo框架安装成功了
      这里可以利用ls -l查看该文件下的目录，或者直接点开你创建的文件夹看该文件的目录

    5. 从本地启动我们的博客，输入：

      hexo s然后他会显示INFO Hexo is running at http://localhost:4000 这么一行代码意思就是从本地4000端口来启动(加粗的地方是你的本地端口号)

    6. 从浏览器里面的地址栏里面输入你的本地地址（例如：http://localhost:4000）

      然后你会看到

      做到这里本地端就算是成功部署了！！！
      当然我们不能把博客仅限于本地浏览，当然还需要部署到远端，那么这个时候我们就是需要github了

    7. 我们接下来去github官网注册一个账户（注意我们一定要想好了在取名字因为我们后面的域名是必须和用户名一样的，博主就吃了这个亏当时没有取一个好听的名字）。GitHub官网（点击进入官网），我们在注册好了之后建立一个新的仓库


      我们在红色框里面填写内容为：（你的用户名）.github.io 这里注意一定要和这个格式保持一直
      然后我们点击create repository创建仓库，其他的选项默认不用管就行，接下来我们把下面的红色框中的地址复制下来

    8. 紧接着我们在我们刚刚建造的博客的文件夹目录下用命令执行窗口部署git的一个插件，在命令执行窗口中输入：

      cnpm install --save hexo-deployer-git
      在安装好后又一红色警告没关系，不影响后面的操作

    9. 然后我们在进行设置_config.yml（在你的自己创建的博客文件夹中的第一个目录下），我们拉到最下面按照我的图片进行更改，第二行的repo填写刚刚我们第7步中复制的你的域名，如下：


      注意git一定要在两个单引号之间

    10. 接下来我们进行最后一步，把我们的文件部署到远端，也就是部署到github的仓库中，我们在创建的博客目录下的命令窗口里输入：
      hexo d

      然后我们就推成功了

      这个就是搭建的利用hexo和github的整个流程了！
